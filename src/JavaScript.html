Data Types in JavaScript

Two types of JavaScript Data type.

  1. Primitive Data Type
  2. Reference Data Type

Primitive Type

Primitive data types are numbers, Booleans, strings, undefined, null, and symbol. Primitive data types are the basic or common data types in JavaScript. Following is the table of primitive data types in JavaScript. 


    string
    Number
    boolean     true / false
    null
    undefined   (if you not assign the value then default show)
    symbol      (new come in ES6)

Non-Primitive Type or Reference Type

Reference in JavaScript are datatypes based on primitive. Like Objects, Arrays, and Functions. Everything is JavaScript is either an Object or Primitive datatype. When we create an object, that value is not directly assigned to the variable. Instead, a reference to that value is what gets set. Variable knows about the location of the object in memory, not the object itself. Following is the table of the reference data type.


    Arrays
    Object Literals
    Functions
    Dates

NaN mean Not a Number

Conditional Statements

Conditional statements are used to perform operations based on some conditions.

Note: if,else,else-if,switch

If Statement

The block of code to be executed, when the condition specified is true.

if (condition) {
        // block of code to be executed if the condition is true
        }

If-else Statement | Control Flow Statement

If the condition for the if block is false, then the else block will be executed.

if (condition) {
	// block of code to be executed if the condition is true
	} else {
	// block of code to be executed if the condition is false
}

Else-if Statement

A basic if-else ladder

if (condition1) {
// block of code to be executed if condition1 is true
} else if (condition2) {
// block of code to be executed if the condition1 is false and condition2 is true
} else {
// block of code to be executed if the condition1 is false and condition2 is false
}

XMLHttpRequest (XHR) Object

The XMLHttpRequest object can be used to exchange data with a web server behind the scenes.This means that it is possible to update parts of a web page, without reloading the whole page.

var xhttp = new XMLHttpRequest();

readyState

In the readyState has 5 steps.

0 Request Not Initialized

1 Server Connection Eastablished

2 Request Received

3 Processing Request

4 Request Finished and Response is Ready

When the Request finished and Server Response is Ready then it return two thing;
status and responseText or responseXML.

Status

http status code : 

 200 : Ok
 403 : Forbidden
 404 : Page Not Found

Send a Request

To send a request to a server, you can use the open() and send() methods of the XMLHttpRequest object: 

xhttp.open("GET|POST","filePath/name.txt",true|false);
xhttp.open("GET","FileName.txt",true);
xhttp.send();

onreadystatechange : property specifies a function to be executed every time the status of the XMLHttpRequest object changes.

readyState : When readyState property is 4 and the status property is 200, the response is ready

responseText :property returns the server response as a text string.

function loadDoc(){
  var xhttp = new XMLHttpRequest();   //creating the Object of XMLHttpRequest()
//xhttp.onready=function(){  same
  xhttp.onreadystatechange=function(){

    if(this.readyState == 4 && this.status == 200){
      //console.log(this.responseText);

    }

  }
  xhttp.open('GET',"test/test.txt",true);
  xhttp.send();
}

The Fetch API interface allows web browser to make HTTP requests to web servers.
No Need of the XMLHttpRequest anymore.
By using Fetch API you can do insert,update,Delete,Read Data from Server.

await

awaitawait makes a function wait for a Promise.
The await operator is used to wait for a Promise . It can only be used inside an async function within regular JavaScript code; however it can be used on its own with JavaScript modules.
If you use the async keyword before a function definition, you can then use await within the function. When you await a promise, the function is paused in a non-blocking way until the promise settles. If the promise fulfills, you get the value back. If the promise rejects, the rejected value is thrown.
