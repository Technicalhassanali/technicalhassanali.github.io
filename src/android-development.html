What is ADB & AVD?

When we create the Android App.Then main steps is to test the android app that work correctly function or not,that user end not show the UI Problem.The testing is necessary to remove the bugs from the app.The Bugs is problem in the android app.

Types of Error

Two types of Error;

  1. compile time error
  2. Run time error

Compile time Error :A compile-time error generally refers to the errors that correspond to the semantics or syntax.

Run time Error : A runtime error refers to the error that we encounter during the code execution during runtime.

ADB : ADB(Android Debug Bridge) : Android Real Device using USB to install APP and test the App

AVD : AVD(Android Virtual Device) : Android Virtual Device is created with different size and test/debug/check UI of the APP.

What is APK?

  • It is a Package file format used by Android Operating system for distribution and installation of Mobile app
  • It means Android Application Package.
  • Similar to executable file .exe in Windows Operating System.
  • The extension of Apk file is .apk.
  • It is installed on Android Operating System.

It consists of

  1. Application code(.dex files)
  2. Manifest file
  3. Assets
  4. Resource files

Example: It just like a zip(winrar) file that collection of multiple file make and show one file.

How to make Apk File?

In Android Studio the build > Build bundle /Apk option on the topbar is used to built the apk file.

Introduction to XML

XML (Extnsible Markup Language).
Language : Set of Instructions that process on the runtime,that is the reason we used Language
Markup : mean changing or formatting the text


  1. It is used for storing and transferring data.
  2. It is a markup language much like HTML but in html the tag is predefined,xml user define tag himselft.The tag define user himself we says the language is extensible.
  3. Unlike HTML, XML is case-sensitive, requires each tag is closed properly, and preserves whitespace.
  4. XML Tags need not to be predefined like HTML known as extensible).
  5. A markup language is a language that annotates text so that the computer can manipulate that text.
  6. Most markup languages are human-readable because the annotations are written in a way to distinguish them from the text itself.

XML in Android

  • We create XML layouts in Android, and later alter them using Java logic.
  • Resources are the additional files and static content an application needs, such as animations, color schemes,layouts, menu layouts.
  • Each layout file must contain one (and only one!) root element(tags).

VIEWS

  • It represents a rectangular area of the screen, and is responsible for displaying information or content, and event handling.
  • Text, images, and buttons are allI Views in Android.

View is a basic building block of UI (User Interface) in android. A view is a small rectangular box that responds to user inputs. Eg: EditText, Button, CheckBox, etc. ViewGroup is an invisible container of other views (child views) and other ViewGroup.

VIEW GROUP

It is essentially an 'invisible container' that holds multiple Views or ViewGroups together, and defines their layout properties.

Common View Groups

  • A List View displays a list of scrollable items.
  • A Grid View displays items in a two-dimensional scrollable grid.
  • A Table Layout groups views into rows and colum.

ROOT VIEWS

It is Root Element of XML Layout file.In starting you take the root tag/element then you can add the code within the root tag/element.

Common View Groups

  • A Linear Layout aligns its contents into a single direction, whether vertical or horizontal.
  • A Relative Layout displays its child content in positions relative to the parent.(Relation b/w Layout)
  • A Frame Layout is a placeholder on a screen thae display only a single view (Fragments)

Android UI Design (XML Layout)

Two ways we design the layout in the android studio;

  1. Create Layout Using Front Drag/Drop Method
  2. Create Layout Using Coding in XML File

XML Layout Using Coding

<?xml version="1.0" encoding="utf-8"?>
<RelativeLayout  
    xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    xmlns:tools="http://schemas.android.com/tools"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    tools:context=".MainActivity">
</RelativeLayout>

What is xmlns? (namespace URl)

  • Uniform Resource Indicator(URl)
  • In computing, it is a string of characters used to identify a name of a resource.
  • Such identification enables interaction with representations of the resource over a network, type the World Wide Web, using specific protocols.

Structure

xmlns:prefix="URI" <!--Syntax-->

URL Namespace

  • XML namespaces are used for providing uniquely name elements and attributes in an XML document.
  • xmins:android describes the android namespace.
  • Also suppose we write our own textview widget with different features compared to android textview, an namespace helps to distinguish between our custome textview widget and android textview widget.
